# PRIVACY PROTOCOL // ZYNTH SECURITY

Last Updated: April 2026

At Zynth, your security is our mission. We collect only what is necessary to protect your assets.

### 1. DATA COLLECTION
We collect scan metadata (URLs, headers, findings) and user credentials (via Supabase Auth) to provide security reporting and dashboard history.

### 2. DATA ENCRYPTION
All data is encrypted in transit using TLS 1.3 and at rest via AES-256 in our Supabase instance. We use Row-Level Security (RLS) to ensure your data is isolated.

### 3. THIRD-PARTY TOOLS
- **AI Engine**: We use Google Gemini to analyze scan findings. No user-identifying data is sent to the AI, only raw scan outputs.
- **Infrastructure**: Our services are hosted on Vercel and Supabase.

### 4. DATA DELETION
You have the "Right to be Forgotten." You can request the permanent destruction of all scan history via the dashboard or by contacting us.

---

### 📡 CONTACT COMMAND
For data-related inquiries, contact the Zynth Security Team via your dashboard support channel.
